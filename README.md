## 0. 개요

**과제 목표**
- 프론트엔드(React + Vite), 백엔드(Express), 데이터베이스(MongoDB Atlas)를 모두 연결한 웹 기반의 Todo 앱을 만든다.  
- 소프트웨어공학 프로세스(요구사항 → 구현 → 테스팅 → 배포)를 파악한다.
- 생성형 AI(ChatGPT, Gemini, Grok, Claude Code)를 적극 활용해 코드 작성 및 디버깅 연습 과정을 익힌다.  

**과제 해결 환경**
- 통합 개발 환경(IDE)으로는 **Google**의 **AntiGravity**를 이용했다.
- 생성형 AI로는 AntiGravity에서 사용할 수 있는 Gemini 3.1 Pro, Claude 4.6 Sonnet, Claude 4.6 Opus 세 모델을 이용했다.
- 보고서는 마크다운 문법으로 Github 리포지토리의 README와 별도의 .md 파일의 형태로 작성되었다. 

**관련 주요 개념**
- **하네스 엔지니어링**(Harness Engineering): 하네스(Harness)는 에이전트가 안정적으로 작업을 수행할 수 있도록 감싸는 전체 환경을, 하네스 엔지니어링은 이 환경을 구축하는 것을 의미한다.
- **바이브 코딩**(Vibe Coding): 소스코드의 생성, 수정, 디버깅 등 개발(코딩) 과정은 전적으로 AI한테 맡기고, 사람(개발자)는 최종 확인만 하는 코딩 방식을 의미한다.

## 2. AI 활용 방법

[안티그래비티 프롬포트 화면(임시)]

- 기본적으로 AntiGravity IDE에서 Claude와 소통하며 실시간으로 소소코드 생성 및 수정 과정을, Gemini와 소통하며 그 외 정보를 얻었다.
- 기능을 구현하는 과정에서 필요했던 의존성 라이브러리 Node.js와 npm의 설치 등의 도움도 Gemini에게 받았다.
- **(중요)** 인공지능 사용 및 Github 커밋 과정에서 .env, API Key 등 **개발 환경이나 개인 정보가 프롬포트에 노출되지 않도록 유의하며** 개발을 진행했다.
	- 얼마 전 실제로 있었던 Claude의 소스 코드 유출 사태도, 프로젝트 배포 시 source map을 포함하는 실수로 인해 발생한 것이였다.

## 3. 느낀 점

- 지난 학기에 수강한 '오픈소스소프트웨어' 과목에서도 Github의 사용법, 바이브 코딩의 개념과 실습 과정을 한 번 봤었는데, 본 과제를 통해 본격적으로 프로젝트를 구성하여 앱을 만들고 배포하는 과정을 거치면서 어느 정도 실전성을 익히는 좋은 계기가 되었다.
- 프로젝트 진행 도중 MongoDB Atlas 백엔드 연결 과정이 조금 복잡하여 시간이 걸렸으나, Gemini의 도움으로 천천히 해결하는 데 성공했고 이 부분에서 성취감을 느꼈다.
- 적극적인 AI 사용으로 며칠 걸릴 것 같았던 프로젝트가 하루 만에 끝나서 좋았지만, 한편으로는 '이런 식의 바이브 코딩 때문에 생길 수 있는 문제점은 없을까?' 하는 걱정을 조금 품으며 프로젝트를 마무리했다.
